omni Engine Technical Manual

Description: This function manages visual properties for elements. It processes requests to get values, set specific values, increase or decrease numerical values, and undo or redo changes. It identifies elements automatically by name and ignores invalid data.

1. Input Targets

The target identifies which element or collection is affected.

  • Single Target: Use a name in quotes (e.g., "box1") or a direct object reference.
  • Group Target: Use an array containing names or references (e.g., ["b1", "b2"]).

2. Single Element Examples

ObjectiveExample CodeResult
Get valueomni("box1", {"width"})Returns single value.
Set valueomni("box1", {"opacity": "0.5"})Sets exact transparency.
Increaseomni("box1", {"left": "+=10"})Adds 10 units.
Decreaseomni("box1", {"left": "-=10"})Subtracts 10 units.
Undoomni("box1").undo()Reverts last change.
Redoomni("box1").redo()Restores undone change.

3. Group Element Examples

ObjectiveExample CodeResult
Group Getomni(["b1", "b2"], {"width"})Returns array of values.
Group Setomni(["b1", "b2"], {"opacity": "0.5"})Sets all listed elements.
Group Increaseomni(["b1", "b2"], {"left": "+=10"})Increases all elements.
Group Decreaseomni(["b1", "b2"], {"left": "-=10"})Decreases all elements.
Group Undoomni(["b1", "b2"]).undo()Reverts the entire collection.
Group Redoomni(["b1", "b2"]).redo()Restores the entire collection.

4. Automatic Processing Rules

  • Data Symmetry: If an array is provided as a target, an array is returned for get requests.
  • Unit Handling: The function identifies existing units (px, em, %) and maintains them automatically.
  • Memory: Each element remembers 500 changes. A new change deletes redo paths.
  • Stability: If instructions are not clear, no action is taken.
