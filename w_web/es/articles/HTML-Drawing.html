¿Qué es la redacción HTML?

Es curioso que preguntes. HTML Redacción, si vamos adelante y revisamos alrededor, es cuando empiezas a inventar algún HTML y utilizas sólo el editor de texto más básico para escribir explícitamente todo el lenguaje de marcado 'intenso'. Eso es lo que descubrirás si buscas la web tratando de entender lo que significa HTML Drafting. ¡No creas todo lo que lees!

¿La redacción HTML no debería ser algo visual? ¿Algo agradable? ¡Algo divertido!

Sí, claro que debería. Por lo tanto, realmente no hay necesidad de pasar todo el tiempo escribiendo todo el código HTML, en todo es un detalle increíble, directamente en un editor de texto. Hay otras formas de hacerlo. Lo que realmente necesitamos es una estrategia para la redacción de documentos HTML.

HTML es extremadamente visual. Todo lo que escribimos en el editor de texto se convierte en algo visual cuando llega al navegador. El navegador mira el código HTML y luego decide cómo debe ser mostrado. Así que, ¿por qué no simplemente vamos adelante y dibujar la mayoría de nuestro HTML directamente dentro del navegador para empezar? ¿No sería mucho más fácil que adivinar cada vez?

¡Sí, suena más fácil! ¿No es así? Así que vamos a seguir adelante y siempre hacerlo de esa manera.


¡Buena llamada! De esa manera podemos escribir casualmente nuestros propios sitios web y todo tipo de otras cosas basadas en HTML.


ESPERA... EL TELÉFONO... ¿HAY ALGÚN PROGRAMA QUE LO HAGA DE ESA MANERA?


Bienvenido a DCKIM.com, casa de redacción HTML.


Nada más, no en ninguna parte de Internet, se llama HTML Drafting. Eso es porque básicamente no existe.

Entonces, '¿por qué es ese el caso?'.


Sólo puedo especular que la razón de este software conspicuo ausente es debido al hecho de que simplemente no hace ningún dinero. La única persona que sería 'enriquecida' por tal software sería el usuario del software. - Gente real.

Con un equipo mundial con expertos en HTML y JavaScript, ninguno de ellos todavía tiene éxito en la producción y publicidad de un programa de redacción HTML para las masas. Los que he visto, no me hacen el corte.


Necesitaba algo más duro que cualquier cosa que hubiera visto. Una especie de retroceso de la era de 1980. Algo que estaba golpeando todas las marcas y revisando todas las cajas.

Necesita tener el teclado como #1, no sólo un post-pensamiento. Por el contrario, necesita ser capaz de ir a Mouse-Only para todo... y eso está bien en marcha ahora.


Este proyecto comenzó lentamente, como fundación, con una selección de los conceptos más sólidos que lo mantienen unidos en su núcleo.

Al igual que con cualquier edificio, si empezamos con las mejores opciones fundamentales, podemos construir eso en algo más interesante.

Necesita ser universal, pero necesita ser preciso y divertido de usar al mismo tiempo. Supongo que ahora sabes cuál es mi concepción de la redacción de HTML.
